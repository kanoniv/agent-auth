# Sudo for Agents - CI/CD Security Kit

> Generated by CEO Plan Review on 2026-03-22
> Status: ACTIVE | Mode: SCOPE EXPANSION

## The Problem

AI agents are being given SSH keys and API tokens like it's 1999. A typical AI-assisted deploy flow:

1. Developer tells Copilot/Devin/Claude to "fix the bug and deploy it"
2. Agent gets a long-lived API token with broad permissions
3. It pushes code, triggers the pipeline, maybe touches infra
4. Nobody has a clean audit trail of what the agent decided vs what the human approved

The "solution" most teams have is vibes. Slightly restricted tokens. Human approval gates that defeat the purpose.

## The Primitive

**"Your AI agents currently have keys. We give them math instead."**

Kanoniv agent-auth provides cryptographic delegation tokens with three properties:

1. **Scope confinement** - `deploy.staging` is mathematically incapable of touching prod. Not policy-blocked, not RBAC-blocked - cryptographically impossible without the root key.
2. **Short-lived authority** - `expires_at` = hard ceiling on compromised tokens. Compare to a leaked AWS IAM key valid until someone notices.
3. **Provenance audit trail** - Every action signed with its delegation chain. "Agent holding token X, issued by human Y at time Z, executed action W within authorized scope."

- **MCP Wrapper**: `kanoniv-auth wrap-mcp <cmd>` - add crypto auth to ANY MCP server

## Demo (the "oh shit" moment)

```python
from kanoniv_auth import delegate, verify

token = delegate(scopes=["deploy.staging"], ttl="4h")

verify(action="deploy.staging", token=token)  # works
verify(action="deploy.prod", token=token)     # raises ScopeViolation
```

The two lines next to each other. That's the moment.

## Scope Violation UX

When an agent tries `deploy.prod` with a `deploy.staging` token:

```
DENIED: scope "deploy.prod" not in delegation

  You have:  ["deploy.staging"]
  You need:  ["deploy.prod"]

  To request escalation:
    kanoniv-auth request-scope --scope deploy.prod --from did:agent:abc...
```

Not just "denied." A teaching moment.

## CLI Commands

```bash
kanoniv-auth init                           # Generate root key
kanoniv-auth delegate --scopes X --ttl 4h   # Issue delegation token
kanoniv-auth verify --scope X --token $T    # Verify token against action
kanoniv-auth sign --action X --token $T     # Sign execution envelope
kanoniv-auth whoami --token $T              # Show DID, scopes, TTL
kanoniv-auth audit <envelope>               # Pretty-print chain as tree
kanoniv-auth serve                          # Start delegation service
kanoniv-auth wrap-mcp <cmd>                 # Wrap MCP server with auth
```

All commands support `--dry-run`.

## Delegation Service

Self-contained Axum + SQLite server (`kanoniv-auth serve`):

- Token lifecycle management (create, verify, revoke, list)
- Auto-renewal before expiry
- Instant revocation propagation
- Built-in `/dashboard` HTML page
- `/health` and `/metrics` endpoints

Free tier = local SQLite. Paid tier = Kanoniv Cloud (Postgres, multi-node, webhooks).

## Demo Workflow

```
Human engineer
    |-- signs delegation token
          scopes: ["build", "test", "deploy.staging"]
          max_ttl: 4h, expires_at: <4h from now>

    Pipeline Orchestrator Agent (receives token)
          |-- sub-delegates to Deploy Agent
                scopes: ["deploy.staging"]  (can only narrow, never widen)

    Deploy Agent executes
          |-- signs execution envelope
                action: deploy, target: staging
                delegation_proof: <full chain>
                result: success

    Audit log = execution envelope hash (verifiable without trusting any system)
```

The prod environment never trusts the agent directly. It only verifies the math on the token chain.

## Key Decisions

- **CLI**: Both Rust (for GH Action) and Python (for devs)
- **Service**: Embedded Axum + SQLite in agent-auth binary
- **PyPI**: New package `kanoniv-auth` (not updating `kanoniv-trust`)
- **Root key**: File for local, OIDC for production CI
- **Repo**: All in `kanoniv/agent-auth` (one repo, one star count)
